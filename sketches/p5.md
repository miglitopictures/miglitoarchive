# Migrating a p5.js sketch into the site

This is a step-by-step guide for taking a normal, standalone p5.js sketch (the kind you'd write in the p5 web editor, with bare `function setup()` / `function draw()`) and converting it into a sketch that works inside `sketches/`, mounted into a project's sidepanel carousel.

---

## 1. Wrap everything in an exported instance-mode function

The site's loader calls `new p5(sketchFn, container)` — this requires **instance mode**, not the global mode the web editor defaults to. Instance mode means every p5 variable and function gets accessed through a `p` object instead of being bare globals.

**Before (global mode, web editor style):**
```js
let w = 600;

function setup() {
    createCanvas(w, w);
}

function draw() {
    background(220);
    ellipse(mouseX, mouseY, 40);
}
```

**After (instance mode, site-ready):**
```js
export default (p) => {

    let w = 600;

    p.setup = () => {
        p.createCanvas(w, w);
    };

    p.draw = () => {
        p.background(220);
        p.ellipse(p.mouseX, p.mouseY, 40);
    };

};
```

Everything moves inside the `(p) => { ... }` function body, and `setup`/`draw`/`mousePressed`/`windowResized`/etc. become assignments (`p.setup = () => {...}`) rather than standalone function declarations.

### Rule of thumb for the `p.` prefix

- **Needs `p.`**: anything that reads from the p5 environment — canvas/window size (`width`, `height`), mouse/keyboard state (`mouseX`, `mouseIsPressed`, `keyIsPressed`), every p5 drawing/math function (`background()`, `ellipse()`, `random()`, `lerp()`, `textWidth()`, `sin()`...), and p5 constants (`LEFT`, `CENTER`, `HAND`, `ARROW`).
- **Doesn't need `p.`**: your own variables (`pg`, `margin`, custom state), and methods called on objects you created yourself — e.g. a graphics buffer from `p.createGraphics()` uses its own methods directly (`pg.noFill()`, `pg.circle()`), since `pg` isn't `p`.

A fast way to catch missed spots: scan for any bare lowercase word that looks like a p5 name sitting outside a `p.`/`pg.`-style call — that's usually a forgotten prefix.

## 2. Move all sketch state inside the exported function

Any `let`/`const` declared at module top-level in a normal sketch needs to move **inside** `export default (p) => { ... }`, not stay outside it.

```js
export default (p) => {

    let pg;
    let margin = 40;
    let lastIndices = [];
    // ...all other sketch state...

    p.setup = () => { ... };
    p.draw = () => { ... };

};
```

**Why this matters**: the sidepanel carousel can mount and unmount the same sketch multiple times in one session (paging away and back, or navigating between projects). If state lived at module scope, it would persist across mounts — old drawings, stale random-history, leftover colors from a previous run. Declaring it inside the exported function means every mount gets fresh state, exactly like a first load.

Helper functions (anything not a p5 lifecycle hook, like a `getUnique()` or `splitTextIntoParts()`) move inside too, as nested functions — this also gives them automatic access to `p` without needing it passed in separately.

---

## 3. Size the canvas to the sidepanel, not the browser window

A normal sketch often does:
```js
p.createCanvas(p.windowWidth, p.windowHeight);
```
`windowWidth`/`windowHeight` are the *entire browser viewport* — inside the site, the sketch only occupies the sidepanel column, not the full window. Using these will make the canvas oversized and get clipped.

Instead, measure the actual mounting element:
```js
p.setup = () => {
    const container = document.querySelector('.sketch-container');
    p.createCanvas(container.clientWidth, container.clientHeight);
};
```

And handle resizing the same way:
```js
p.windowResized = () => {
    const container = document.querySelector('.sketch-container');
    p.resizeCanvas(container.clientWidth, container.clientHeight);
};
```

**If the sketch uses a secondary graphics buffer** (`p.createGraphics(...)`), remember `resizeCanvas` only resizes the main canvas — it has no idea a second buffer exists. Recreate the buffer inside `windowResized` too, mirroring whatever `setup()` does:
```js
p.windowResized = () => {
    const container = document.querySelector('.sketch-container');
    p.resizeCanvas(container.clientWidth, container.clientHeight);

    pg = p.createGraphics(p.width - margin * 2, p.height - margin * 2);
    pg.noFill();
    p.background(200);
    pg.background(220);
};
```
Note this clears whatever was drawn on the buffer — worth deciding if that tradeoff is acceptable for a given sketch, or whether resizing should be avoided/handled differently for something with precious user-drawn content.

## 4. Watch for the p5 2.x space-measurement gotcha

If a sketch measures text **character by character** (rather than whole words/strings at once), be aware: p5 2.0+ trims leading/trailing whitespace before measuring text width. A lone space character, measured on its own, is *entirely* leading/trailing whitespace, so `p.textWidth(' ')` returns `0` — spaces collapse and words run together. This did not happen in p5 1.x, where `textWidth` measured whatever string it was given literally.

**Fix**: don't measure a lone space directly. Sandwich it between non-space sentinel characters, measure that, and subtract the sentinels' own width:
```js
function measureChar(char) {
    if (char !== ' ') {
        return p.textWidth(char);
    }
    const sentinel = '|';
    return p.textWidth(sentinel + ' ' + sentinel) - 2 * p.textWidth(sentinel);
}
```
Since this value never changes for a fixed font/size, cache it once instead of recalculating every frame:
```js
let cachedSpaceWidth = null;

function measureChar(char) {
    if (char !== ' ') return p.textWidth(char);
    if (cachedSpaceWidth === null) {
        const sentinel = '|';
        cachedSpaceWidth = p.textWidth(sentinel + ' ' + sentinel) - 2 * p.textWidth(sentinel);
    }
    return cachedSpaceWidth;
}
```
This only matters for sketches doing per-character measurement — sketches that call `textWidth()` on whole strings/words are unaffected, since a real word never has whitespace at *both* ends.

---

## 5. Export the sketch

Once the sketch is fully wrapped, the file's outermost shape is:
```js
export default (p) => {
    // all state, p.setup, p.draw, p.mousePressed, p.windowResized,
    // and any helper functions, live in here
};
```
Save it into `sketches/`, named for the project it belongs to (e.g. `sketches/painting-matter.js`).

Then reference it from the project's entry in `projects.js`:
```js
sidepanel: [
    { type: "about", text: { en: "...", pt: "..." } },
    { type: "p5", src: "../sketches/your-sketch-name.js" },
],
```

---

## 6. Test independently before wiring it into the site

Because the exported function is just a plain instance-mode sketch, it works identically outside the site's router:

**In the p5 web editor**: paste in the same `p.`-prefixed code, and add one line at the bottom to actually run it there:
```js
let mySketch = (p) => {
    p.setup = () => { ... };
    p.draw = () => { ... };
};
new p5(mySketch);
```
To move it into `sketches/` afterward: delete the `new p5(mySketch)` line, and change `let mySketch = (p) => {` to `export default (p) => {`.

**Locally, via `sketches/_test.html`** (not linked from the live site — just a harness):
```html
<!DOCTYPE html>
<html>
<head><script src="https://cdn.jsdelivr.net/npm/p5@1.9.4/lib/p5.min.js"></script></head>
<body style="margin:0">
<script type="module">
  import sketch from './your-sketch-name.js';
  new p5(sketch);
</script>
</body>
</html>
```
Open this through `dev-server.py` to iterate on a sketch with zero dependency on `main.js` or the router.

## Checklist

- [ ] Wrapped in `export default (p) => { ... }`
- [ ] Every p5 global/function call has a `p.` prefix
- [ ] All sketch state (`let`/`const`) and helper functions moved inside the exported function
- [ ] Canvas sized from `.sketch-container`'s `clientWidth`/`clientHeight`, not `p.windowWidth`/`p.windowHeight`
- [ ] `p.windowResized` implemented, including re-creating any secondary graphics buffers
- [ ] If measuring text character-by-character: space-width handled with the sentinel technique, cached
- [ ] Sketch file saved in `sketches/`, referenced via a `{ type: "p5", src: "..." }` entry in the project's `sidepanel` array
- [ ] Verified standalone (web editor and/or `_test.html`) before relying on it inside the site
